# 公众号爆款文章创作大师

基于《AI 扔下的生肉，谁来煮？》真实创作过程提炼的完整方法论。

## 核心理念

**爆款文章 = 多篇融合 + 人设注入 + 坚决利他 + 文笔润色 + 对抗式审核**

不是文采，是真诚。
不是说教，是共鸣。
不是展示，是对话。

## 五大核心要素

### 1. 多篇爆款融合

**目标**：从 3-5 篇相关文章中提取核心观点，重新组合

**操作步骤**：

```bash
# Step 1: 获取文章内容
curl -s "公众号URL" -H "User-Agent: Mozilla/5.0" > /tmp/article1.html

# Step 2: 提取正文
python3 << 'EOF'
import re, html
with open('/tmp/article1.html', 'r', encoding='utf-8') as f:
    content = f.read()
match = re.search(r'<div class="rich_media_content[^>]*>(.*?)</div>\s*<script', content, re.DOTALL)
if match:
    text = re.sub(r'<[^>]+>', '\n', match.group(1))
    text = html.unescape(text)
    print(text)
EOF
```

**提取清单**：
- [ ] 核心观点（每篇 1-3 个）
- [ ] 具体案例（人物、数据、故事）
- [ ] 金句（可引用的）
- [ ] 数据支撑（数字、百分比、对比）
- [ ] 独特概念（如"生肉证明"、"阻抗失配"）

**融合原则**：
- ✅ 提取观点，不是复制文字
- ✅ 寻找可以连接到个人经历的点
- ✅ 用自己的话重新表达
- ❌ 不要直接拼接多篇文章

**案例**：
```
文章 1：陶哲轩演讲
  提取：证明稀缺 → 证明过剩、"生肉证明"概念
  
文章 2：重磅发声
  提取："阻抗失配"、Liam Price 案例、从证明者到品鉴师
  
文章 3：互联网已死
  提取：Agent 是用户、算力是特权、Token 消耗决定进化速度
  
融合策略：
  用陶哲轩的故事作为引子（10%）
  用 Andy 自己的 AI 酋长账号经历作为主线（70%）
  用算力特权作为延伸思考（20%）
```

### 2. 注入个人人设信息

**目标**：让文章成为"只有你能写出来"的作品

**人设信息收集**：

创建人设档案 `/tmp/persona.md`：
```markdown
# 个人背景
- 教育：数学专业出身
- 经历：大学数学建模比赛，学院办公室熬到凌晨 3 点
- 职业：腾讯/阿里 AI 团队背景
- 创业：大量创业经历，不断突破边界

# 正在做的事
- AI 酋长案例账号：视频版 AI 百科全书
- 醒觉社：探索 AI 共生时代的人类进化

# 具体数据（真实案例）
- AI 酋长账号：前 20 期 7-10 天/期 → 第 40 期 1 天/期
- 粉丝增长：月涨 2000 → 8000
- 第 48 期实验：播放量少 30%，但"感觉回来了"
- "火花"文件夹：127 条灵感记录

# 个人特质
- 喜欢折腾、探索、验证
- 不断突破边界
- 数学思维 + 产品思维 + 创业思维
```

**注入原则**：

1. **用真实案例替代抽象观点**
   - ❌ "AI 让很多事情变快了"
   - ✅ "AI 酋长账号从 7-10 天/期变成 1 天/期，效率提升 10 倍"

2. **用具体数字替代模糊描述**
   - ❌ "粉丝增长很快"
   - ✅ "粉丝从每月涨 2000 到每月涨 8000"

3. **用真实情感替代文学化场景**
   - ❌ "凌晨三点，我在思考人生"
   - ✅ "凌晨三点，我盯着 Claude 生成的脚本，手指悬在发布按钮上方，不敢按下去"

4. **用顿悟时刻替代平铺直叙**
   - ❌ "我意识到了一个问题"
   - ✅ "凌晨 2:47，我跳起来，指着白板说：'你们看，如果我们把这个变量替换成这个，整个模型就通了！'"

**人设注入检查清单**：
- [ ] 有没有作者自己的真实案例？（至少 3 个）
- [ ] 每个案例都有具体数字吗？
- [ ] 有没有 before/after 对比？
- [ ] 有没有真实的情绪波动？（焦虑、兴奋、困惑、顿悟）
- [ ] 读者能感受到"这个人"吗？

### 3. 坚决利他

**目标**：70% 讲"怎么办"，30% 讲"是什么"

**利他原则**：

读者不缺观点，缺方法。
读者不缺焦虑，缺行动方向。
读者不缺故事，缺可操作的步骤。

**文章结构模板**：

```
30% - 现象描述（用自己的故事）
  ├─ 开头：冲击力场景（100 字）
  ├─ 困惑：我遇到了什么问题（500 字）
  └─ 探索：我做了什么尝试（800 字）

70% - 方法论（可操作）
  ├─ 5 个生存法则（每个 300-400 字）
  │   ├─ 法则描述
  │   ├─ 为什么重要
  │   └─ 具体操作（必须有）
  │
  └─ 3 件可以从明天开始做的事（每个 150 字）
      ├─ 第一件事：具体行动
      ├─ 第二件事：具体行动
      └─ 第三件事：具体行动
```

**"具体操作"模板**：

每个方法都必须有"具体操作"小节：

```markdown
### 法则 1：用 AI 做"脚手架"，不做"成品"

现在我用 AI 的方式是：
- 让它帮我整理资料、提取要点
- 让它生成初稿，但我会重写至少 50%
- 让它提供 3-5 个角度，我选一个深挖

**具体操作**：每次让 AI 生成内容后，我会问自己三个问题：
- 这里面有我的独特视角吗？
- 这里面有我的真实经验吗？
- 如果去掉署名，别人能认出这是我写的吗？

如果答案是"否"，我就重写。
```

**利他检查清单**：
- [ ] 每个方法都有"具体操作"吗？
- [ ] 读者看完知道下一步该做什么吗？
- [ ] 有没有"从明天开始做的 3 件事"？
- [ ] 方法是否可验证？（读者做了能看到结果）
- [ ] 有没有给出具体的标准/数字？（如"重写至少 50%"、"总分低于 24 分就重做"）

### 4. 文笔润色（画面感+文学感）

**目标**：让文章有画面感、能共鸣、像人在说话

**润色原则**：

#### 原则 1：用画面替代抽象

❌ **抽象描写**：
"我很焦虑，不知道该怎么办。"

✅ **画面感描写**：
"凌晨三点，我盯着 Claude 刚生成的 2000 字脚本，手指悬在'发布'按钮上方。我读了两遍，挑不出毛病。但我就是不敢按下去。"

#### 原则 2：用对话替代叙述

❌ **叙述**：
"我突然意识到了问题的关键。"

✅ **对话**：
"凌晨 2:47，我跳起来，指着白板说：'你们看，如果我们把这个变量替换成这个，整个模型就通了！'"

#### 原则 3：用细节替代形容词

❌ **形容词堆砌**：
"那是一个非常艰难的夜晚，我们非常努力地工作。"

✅ **具体细节**：
"我们在白板上写满了公式，擦掉，重写，再擦掉。咖啡喝了六杯，草稿纸用了一摞。"

#### 原则 4：用数字替代模糊描述

❌ **模糊**：
"写了很久，只写出一点点。"

✅ **具体**：
"写了一个小时，只写出 300 字。删了，重写。又写了一个小时，500 字。"

#### 原则 5：变化句子长度

❌ **单调节奏**：
"我开始写作。我发现很困难。我写得很慢。我感到焦虑。"

✅ **节奏变化**：
"我开始写作。写了一个小时，只写出 300 字。我发现自己的'写作肌肉'已经萎缩了。"

**润色检查清单**：
- [ ] 开头有画面感吗？（能让人"看到"场景）
- [ ] 有没有具体的对话？
- [ ] 有没有具体的细节？（时间、地点、数字、动作）
- [ ] 句子长短有变化吗？
- [ ] 读起来像人在说话吗？

**去 AI 感检查**：

使用 `humanizer` skill 检查：
```bash
skill_view(name='humanizer')
```

删除这些 AI 常用词：
- ❌ 至关重要、深入探讨、值得注意的是
- ❌ 首先...其次...最后...
- ❌ 综上所述、总而言之
- ❌ 让我们深入探讨、接下来我们来看看

### 5. 对抗式内容审核

**目标**：用毒舌参谋的视角，找出致命问题

**审核流程**：

#### Step 1: 调用对抗式审核

```bash
# 方法 1：使用 adversarial-review skill
skill_view(name='adversarial-review')

# 方法 2：使用 delegate_task 模拟参谋角色
delegate_task(
  goal="作为毒舌内容参谋，用 5 维度攻击框架严格审核文章",
  context="
你是反方。你的工作不是夸文章，是找漏洞、找软肋、找读者会弃读的地方。

必须覆盖 5 个维度：
1. 标题吸引力 - 能让人点开吗？给出 3 个替代标题
2. 结构逻辑 - 骨架稳不稳？开头抓人吗？结尾有力吗？
3. 数据支撑 - 观点有数据/案例支撑吗？哪里是空话？
4. 读者视角 - 读者看完有收获吗？会不会弃读？有没有说教感？
5. 篇幅比例 - 每个部分篇幅合理吗？哪里写多了哪里写少了？

特别关注：
- 作者的个人经历是否真实可信？还是为了写而写？
- 情感是否真诚？还是在堆砌文学化的场景？
- 人设信息是否自然？还是像广告？
- 有没有'只有作者能说出来'的独特洞察？
- 读者会不会觉得'这人真的懂'？

文章内容在 /tmp/article.md
  ",
  toolsets=["file", "terminal"]
)
```

#### Step 2: 5 维度攻击框架

**维度 1：标题吸引力（X/10）**
- 能让人在信息流里停下来点吗？
- 有冲突感、悬念感吗？
- 给出 3 个替代标题建议

**维度 2：结构逻辑（X/10）**
- 开头抓人吗？前 100 字如果不抓人，读者就走了
- 核心论点清晰吗？
- 有没有逻辑跳跃？
- 结尾有力量吗？

**维度 3：数据支撑（X/10）**
- 核心观点有数据/案例佐证吗？
- 有没有"光说结论不给证据"的地方？
- **作者自己的案例在哪里？**（最重要）
- 有没有空话？（如"认知差是一百倍"）

**维度 4：读者视角（X/10）**
- 读者看完有什么收获？
- 哪里会让人弃读？
- 有没有说教感？
- 读者能做什么？（可操作性）
- 会不会分享到朋友圈？

**维度 5：篇幅比例（X/10）**
- 开场是否太长？
- 核心论点是否太薄？
- 哪些部分可以砍掉一半？
- 哪些部分需要展开？

#### Step 3: 评分标准

- **9-10 分**：✅ 直接通过，可以发布
- **8 分**：✅ 通过（可带轻微修改建议）
- **6-7 分**：🔁 打回，需要修改
- **≤5 分**：🔁 重写

#### Step 4: 最致命的 3 件事

审核报告必须给出"最致命的 3 件事（笔杆子必须解决的）"：

```
1. [问题 1]
   为什么致命：[原因]
   如何解决：[具体建议]

2. [问题 2]
   为什么致命：[原因]
   如何解决：[具体建议]

3. [问题 3]
   为什么致命：[原因]
   如何解决：[具体建议]
```

**案例：第一版审核结果**
```
综合预判分：6/10 — 打回重写

最致命的 3 个问题：

1. 观点都是二手的，Andy 自己的一手经验在哪里？
   为什么致命：读者会怀疑"这人真的懂吗？还是在蹭热点？"
   如何解决：补上 AI 酋长账号的具体数据、醒觉社的真实案例

2. 70% 在讲"是什么"，只有 5% 在讲"怎么办"
   为什么致命：读者看完只会焦虑，不知道下一步该干什么
   如何解决：倒过来，30% 讲现象，70% 讲方法，给出可操作的步骤

3. 醒觉社广告位太刻意，口号式结尾太自我感动
   为什么致命：读者觉得在看广告，不是在看文章
   如何解决：删掉所有"我在做醒觉社的时候"，结尾用具体行动号召
```

## 常见陷阱与避坑指南

### 陷阱 1：观点都是二手的

**症状**：
- 全文都在转述别人的故事
- 没有作者自己的一手经验
- 读者会怀疑："这人真的懂吗？"

**解决方案**：
1. 列出作者的 5 个真实经历
2. 每个经历要有：时间、地点、具体数据、情感波动
3. 至少 3 个经历要深入展开（500+ 字）

### 陷阱 2：讲"是什么"太多，讲"怎么办"太少

**症状**：
- 70% 在讲"AI 很快，世界在变"
- 只有 5% 在讲"你该怎么做"
- 读者看完焦虑，但不知道下一步

**解决方案**：
1. 倒过来：30% 讲现象，70% 讲方法
2. 每个方法都要有"具体操作"小节
3. 结尾给出"从明天开始做的 3 件事"

### 陷阱 3：情感不真实

**症状**：
- 凌晨三点的场景太文学化
- 情绪描写像在表演
- 读者感受不到"这个人"

**解决方案**：
1. 用具体细节替代抽象描写
2. 用对话替代叙述
3. 用数字替代形容词

### 陷阱 4：广告位太刻意

**症状**：
- "我在做醒觉社的时候"出现 3 次
- 每次提到产品都很生硬
- 读者觉得在看广告

**解决方案**：
1. 产品名称最多出现 2 次
2. 删掉所有"我在做 XX 的时候"句式
3. 结尾一笔带过，不要反复强调

### 陷阱 5：深度思考文的特殊陷阱

**当用户要求"深度思考文"而非"工具指南"时，避免以下错误：**

**症状**：
- 用 10 个"朋友告诉我"案例，但无一有可验证信息（姓名、项目、时间）
- 作者只有"观察"没有"实践"，缺乏深度使用经验
- 立场摇摆：既说 AI 好又说 AI 坏，读者不知道作者立场
- 结尾逃避："我不知道答案"+"你自己想"
- 想说的太多（进化、失去、获得、方法、哲学），每个都没说透

**解决方案**：
1. **案例真实性**：
   - 删减"朋友告诉我"到 2-3 个最有冲击力的
   - 增加作者自己的 3 个深度实践案例（具体工具、项目、数据、时间）
   - 提供可验证信息（不是"127 条灵感 70% 跨领域"这种过于精确的数字）

2. **明确立场**：
   - 不要"既说好又说坏"，要有明确的价值判断
   - 建议立场："拥抱 AI，但不依赖 AI"
   - 删掉所有自相矛盾的地方

3. **结尾有力**：
   - 删除"我不知道答案"+"你自己想"的逃避式结尾
   - 给出明确立场和行动指引
   - 用金句收尾，但金句必须有实质内容（不是"进化不是目的，选择才是"这种空话）

4. **聚焦深度**：
   - 砍掉 30% 的内容
   - 把剩下的 70% 说得更深、更透、更真实
   - 如果是哲学追问，至少 600 字展开，不要点到即止

5. **平衡正负情绪**：
   - "失去什么"不要超过 600 字（避免贩卖焦虑）
   - "获得什么"至少 800 字（给读者希望）
   - 比例失衡会让读者觉得作者只会批判，不会建设

6. **数据来源**：
   - 所有数据必须有来源或删除（如"注意力 12 秒→8 秒"）
   - 个人数据不要过度使用（45+ 项目、127 条灵感）会显得炫耀
   - 关键观点必须有数据支撑，不能只有主观感受

**深度思考文 vs 工具指南的区别**：
- 工具指南：70% 方法论 + 30% 现象，结尾给"3 件明天就能做的事"
- 深度思考文：40% 现象 + 30% 深度分析 + 20% 方法 + 10% 哲学追问，结尾给明确立场而非行动清单

**关键原则**：深度思考文的价值在于"让读者思考得更深"，而不是"让读者立刻行动"。但"让读者思考"≠"不给答案"，作者必须有明确立场。

## 支持文件

### references/deep-thinking-article-pitfalls.md
深度思考文的特殊陷阱与解决方案。当用户要求"深度思考文"而非"工具指南"时，必须参考此文件。

核心区别：
- 工具指南：70% 方法论，结尾给行动清单
- 深度思考文：40% 现象 + 30% 深度分析 + 20% 方法 + 10% 哲学追问，结尾给明确立场

关键陷阱：
1. 案例真实性危机（10 个"朋友告诉我"但无可验证信息）
2. 作者实践经验不足（只有观察没有实践）
3. 立场摇摆（既说好又说坏）
4. 结尾逃避（"我不知道答案"+"你自己想"）
5. 想说的太多，每个都没说透
6. 正负情绪失衡（贩卖焦虑）
7. 数据真实性存疑（数字太精确反而不真实）

### references/iterative-polishing-workflow.md
深度长文的迭代打磨工作流。记录了《人类正在进化成三体人》从 4200 字 4.5 分到 2650 字 8.5+ 分的完整过程（5 次迭代）。

核心经验：
1. **对抗式审核是核心**：每次迭代都必须有 5 维度审核，找出最致命的 3 个问题
2. **方法论必须前置**：在 1500 字处出现，不要等到 2500 字
3. **哲学追问不能超过 15%**：超过会成为弃读高峰区，必须压缩到 300 字以内
4. **篇幅控制黄金比例**：理想字数 2500-3000 字，超过 3000 字必须有足够的"喘息空间"
5. **品牌露出最多 2 次**：超过会显得像广告，削弱利他感
6. **用户说"继续优化"**：不要问改哪里，直接找出所有问题一次性改完
7. **用户说"篇幅要短"**：大刀阔斧删减 20-30%，保留精华删除冗余

迭代终止条件：对抗式审核 ≥8 分 + 你自己会转发到朋友圈

## 总结

**爆款公式**：

```
爆款 = 多篇融合（3-5篇） 
     + 人设注入（真实案例+数据） 
     + 坚决利他（70%方法论） 
     + 文笔润色（画面感+文学感） 
     + 对抗式审核（≥8分才发布）
```

**记住**：
- 真诚 > 文采
- 方法 > 观点
- 删减 > 添加
- 审核 > 直觉

**最重要的一条**：
如果你自己都不想分享到朋友圈，那就还没写好。
